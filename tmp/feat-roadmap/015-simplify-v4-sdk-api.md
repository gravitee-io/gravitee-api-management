# 015 - Simplify V4 SDK Integration

## Goal

Refresh the V4 SDK to include enhanced JSON Patch serialization (fixing the `jsl error [json_parse_error]: missing field 'op'` issue) and a unified entrypoint (Enum + Static Factory Method). Then, simplify the backend integration to leverage this new SDK.

## Description

Currently, `LlmEngineServiceImpl` relies on string mapping (`"Api"`, `"v4/Flow"`) to functional references (`Api::generate`). This logic will be removed and replaced by using the new `V4Component` Enum (or equivalent) generated by the `json-schema-llm` project.
The frontend should also be tested to ensure the payload works with the simplified API.

## Checklist

- [ ] Wait for `json-schema-llm` to be fixed upstream (Jackson `writerFor()` serialization and unified Enum / Generator entrypoint).
- [ ] Vend the updated `v4-0.1.0-ALPHA.jar` locally to `~/.m2` and copy to `distribution/lib/ext/`.
- [ ] Update POST `/management/v2/environments/{envId}/ai/generate` to accept the new V4 SDK enum.
- [ ] Remove the manual `ComponentInvoker` map for V4 models in `LlmEngineServiceImpl`.
- [ ] Test the Console UI to ensure V4 component generation (e.g., `Api`, `Flow`) works flawlessly.
