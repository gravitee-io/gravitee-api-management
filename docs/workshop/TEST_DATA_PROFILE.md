# Test Data Profile for Analytics Dashboard

This document describes the synthetic dataset generated by `scripts/generate-test-data.sh` and what to expect in each dashboard widget.

## What the script generates

- Creates (or reuses) one v4 HTTP Proxy API:
  - Name: `Workshop Analytics V4 Test API` (configurable)
  - Path: `/workshop-analytics-v4-test` (configurable)
  - Backend target: `https://httpbin.org` (configurable)
- Requests deployment and starts the API.
- Sends **120 requests** with controlled variation:
  - HTTP methods: `GET`, `POST`, `PUT`, `DELETE`
  - Paths/status outcomes:
    - `/get`, `/post`, `/put`, `/delete` -> `200`
    - `/status/201` -> `201`
    - `/status/400` -> `400`
    - `/status/404` -> `404`
    - `/status/500` -> `500`
  - Payload sizes:
    - small JSON payloads on selected `POST`
    - large JSON payloads (~12KB blob) on selected `PUT`
  - Time-series shaping:
    - traffic sent in 10 bursts with short sleeps between requests and larger sleeps between bursts

### Total volume

- `404`: 16.7%
- `500`: 16.7%

## Expected dashboard behavior

### Stats cards (US-07)

- **Total Requests**: should increase and be non-zero for the selected timeframe.
- **Avg Gateway Response Time**: should be non-zero after traffic lands.
- **Avg Upstream Response Time**: should be non-zero after traffic lands.
- **Avg Content Length**: should be non-zero and higher than trivial values due to mixed small/large payloads.

### HTTP status pie (US-08)

- Should show slices for `200`, `201`, `400`, `404`, `500`.
- `200` slice should be the largest.
- Remaining status slices should be approximately equal.

### Response status over time / response time over time (legacy widgets)

- Should show visible non-flat patterns (bursty shape) because traffic is intentionally spaced over time.
- Multiple time buckets should be populated, not just a single spike.

### Page-level empty state (US-09b)

- Should **not** be displayed after running the script and waiting for indexing.

## Elasticsearch verification

The script checks:

- `GET http://localhost:9200/gravitee-v4-metrics-*/_count`

Interpretation notes:

- Count should increase after the run.
- Absolute count can be greater than 120 if:
  - prior runs already inserted data
  - other traffic exists in the environment

## Practical usage

- Run from repository root:
  - `./scripts/generate-test-data.sh`
- Default management credentials are `admin/admin` (override if needed):
  - `MGMT_USERNAME=... MGMT_PASSWORD=... ./scripts/generate-test-data.sh`
- Token auth is also supported:
  - `MGMT_TOKEN=... ./scripts/generate-test-data.sh`
